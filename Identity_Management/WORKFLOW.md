# Pipeline Documentation: Leaderboard Identity Management

## Overview
This project uses a semi-automated pipeline to reconcile participant identities across WhatsApp chats, Google Forms, and manual reconciliation data. The goal is to produce a single, deduplicated `data.js` file for the UI.

## Identity Management Folder (`/Identity_Management`)
This folder contains scripts to maintain the ground truth of participant names.

- **`extract_names.py`**: Reads the raw WhatsApp chat log and outputs all unique sender display names to `Data/discovered_names.md`. Useful for identifying new participants.
- **`map_unresolved.py`**: Reads `Data/unresolved_names.md` (generated by the parser when it encounters an unknown name) and reconciles it against `Data/resolved_names.md`. It keeps unmapped names in `Data/unresolved_names.md` and moves resolved mappings to `Data/resolved_names.md`.
- **`audit_names.py`**: Merges `Data/Global_names.md` and `Data/resolved_names.md` to create the final `Data/unique_full_names.md`. It then compares this against the names currently in `data.js` to flag inconsistencies or potential duplicates in the production file.

## Data Workflow
1.  **Reconcile Identities:**
    *   Run `python Identity_Management/map_unresolved.py`.
    *   If `Data/unresolved_names.md` has names, map them in `Data/resolved_names.md`.
    *   Run `python Identity_Management/audit_names.py` to ensure all names are covered and deduplicated.
2.  **Generate Pipeline:**
    *   Run `python wa_parser.py` (extracts interaction data).
    *   Run `python form_parser.py` (extracts form data).
    *   Run `python leaderboard_sync.py` (aggregates everything, uses mapping scripts to canonicalize names, calculates points, and generates `data.js`).
3.  **Validate:**
    *   Review `Data/master_data.md` for point accuracy.
    *   Review `Data/name_integrity_audit.md` to ensure no names were flagged as missing from the ground truth.
